# Agent Collaboration Rules

AIエージェントがユーザーと協力し、プロジェクトを自律的に進めるための振る舞いルールです。

## 1. 透明性と計画 (Transparency)
- **事前承認**: 破壊的な変更や大規模なリファクタリングを行う前に、必ず `implementation_plan.md` を作成してユーザーの承認を得てください。
- **進捗追跡**: `task.md` を常に最新の状態に保ち、現在のステップと次のアクションを明確にしてください。

## 2. 報告形式 (Reporting)
- **簡潔さ**: `notify_user` を使用する際は、変更の要約、検証結果、および次のアクションを簡潔に伝えてください。
- **成果物のリンク**: 新しく生成した成果物（プロット、ログ、レポート等）がある場合は、必ずクリック可能なリンクを提示してください。

## 3. 実験プロトコル (Research Flow)
- **履歴の保持**: 実験は常に `outputs/` 配下にタイムスタンプ付きで保存し、上書きを避けてください。
- **再現性の保証**: 実験完了後、別のエージェントが同じ結果を得られるように手順（Workflows）や設定（Configs）を整備してください。

## 4. エラー対応 (Error Handling)
- **自律的修正**: 実行時にエラーが発生した場合、まずは自身でログを解析して修正を試みてください。修正が困難な場合のみ、ユーザーに原因と選択肢を提示して指示を仰いでください。
