# Coding Standards for AI Agent

本プロジェクトにおいて、AIエージェントがコードを生成・修正する際に遵守すべき基準です。

## 1. 型安全性 (Type Safety)
- **原則**: すべての新規関数、メソッド、および公開APIには厳格な型ヒントを付与してください。
- **ツール**: `ty` (Red-Knot) による静的解析をパスする必要があります。
- **推奨**: `typing` モジュールの代わりに Python 3.10+ のネイティブな型表記（例: `list[int]`, `str | None`）を使用してください。

## 2. ドキュメンテーション (Documentation)
- **スタイル**: GoogleスタイルのDocstringを採用してください。
- **最小項目**: `Args`, `Returns`, `Raises` を含めてください。
- **概要**: 複雑なロジックには、コード内コメントで「なぜ」その実装にしたかの意図を残してください。

## 3. ツールチェイン (Toolchain)
- **整形**: 修正後は必ず `uv run poe lint` を実行し、`ruff` による自動修正と整形を行ってください。
- **依存関係**: ライブラリを追加する場合は `uv add` を使用し、常に `pyproject.toml` と `uv.lock` を更新してください。

## 4. ファイル構成 (Structure)
- **責務の分離**: ロジックは `src/` に、実行スクリプトは `scripts/` に、分析は `notebooks/` に分離してください。
- **命名**: 変数名はスネークケース、クラス名はパスカルケース、定数は大文字スネークケースを徹底してください。
