# プロジェクト概要: AI-Driven Research Template

## コンセプト
本プロジェクトは、**AIエージェント（Antigravity, Cursor, GitHub Copilot等）の力を最大限に引き出し、研究の試行錯誤（Iteration）を極限まで高速化する**ためのPythonプロジェクトテンプレートです。

## 解決したい課題
- 研究の反復が遅く、仮説検証のコストが高い
- 環境差異や手順の属人化により再現性が崩れる
- AIエージェントがコンテキストを誤解し、不要な往復が発生する

## 提供する価値
- **高速な試行錯誤**: `uv`/`ruff`/`ty` による開発支援と、`marimo` によるリアクティブな分析環境。
- **再現性の担保**: 実験の入力/出力/ログが定型化され、`uv.lock` により環境が完全固定。
- **タスク実行の簡略化**: `poethepoet` により、複雑な実験コマンドを短い単語で実行可能。
- **AIとの協調最適化**: 構造とルールが明文化され、エージェントが判断しやすい。

## 核心となる哲学
1. **Astral First**: `uv`, `ruff`, `ty` といったAstral社のRust製ツールを用いることで、管理・解析の待ち時間をゼロに近づけます。
2. **Reactive Analysis**: `marimo` を採用し、コードと出力の不整合（ノートブックの混沌）を排除します。
3. **AI-Friendly Architecture**: 機械（AI）が読みやすく、修正の影響範囲を即座に理解できるクリーンで説明的な構造を採用します。
4. **Reproducibility & Automation**: `uv` のロックファイルと `poethepoet` によるタスク化により、誰でも同じ手順で実験を再現・自動化できます。

## 主な機能
- **標準化されたディレクトリ**: 役割が衝突しない明確な責務分離
- **CLI駆動の実験実行**: `uv run` を前提にしたスクリプト設計
- **成果物の一元管理**: `outputs/`/`data/interim/` に結果を集約
- **AIスキル拡張**: `.agent/skills/` によるモジュール型の専門能力
- **ドキュメント主導**: 運用ルールを `docs/` に明文化

## 対象ユーザー
- 最新のPythonツールを使いこなし、研究を効率化したい研究者
- AIエージェントにコード作成や実験の実行を任せたい開発者

## 想定ユースケース
- アブレーションやハイパーパラメータ探索の高速化
- 複数エージェントによる並行実験の管理
- 実験結果の可視化と再実行の自動化

## 成果物とドキュメント
- `docs/overview.md`: 全体像と設計思想
- `docs/architecture.md`: 構造と責務分離
- `docs/pdca_execution.md`: 実行プロトコル
- `docs/ai_collaboration.md`: エージェント協調ルール
- `docs/roadmap.md`: 実装ステップ

## 目標
- 環境構築が1分以内に完了すること
- 全ファイルへの型チェックとLintが数秒以内に終わること
- AIエージェントがプロジェクトの構造を即座に理解し、正しい提案ができること

## 非目標
- 特定分野の研究フレームワークやモデルを提供すること
- 大規模なデータ管理基盤や配布システムの実装
- UI中心のワークフロー（本テンプレートはCLI中心）
